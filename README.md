# RAG Chatbot – Semantic Retrieval API

A RAG (Retrieval-Augmented Generation) chatbot with a "Reasoning and Retrieval" pipeline: semantic chunking, parent-child hierarchy, hybrid search, and re-ranking.
## Stack

- **Framework:** LangChain
- **Vector store:** Qdrant (embedded, local; persisted under `qdrant_db/`)
- **Embeddings:** OpenAI `text-embedding-3-small`
- **Re-ranker:** `cross-encoder/ms-marco-MiniLM-L-6-v2` (SentenceTransformers)
- **LLM:** OpenAI (e.g. `gpt-4o-mini`) for answer generation and for metadata (summary, target_question) during ingestion

## Setup

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Create a `.env` file (you can copy from `.env.example` if present):

```
OPENAI_API_KEY=sk-your-key
```

## Project structure

| File | Description |
|------|-------------|
| `config.py` | Settings: models, paths, chunk sizes, retrieval parameters |
| `ingest.py` | Load PDF/DOCX, SemanticChunker, parent-child chunks, metadata (summary, target_question), store in Qdrant |
| `retriever.py` | Query expansion, hybrid search (vector + Qdrant native sparse), re-ranking, parent context retrieval |
| `app.py` | FastAPI: `/ask` with grounded answers and citations |
| `streamlit_app.py` | Streamlit demo UI for ingest and chat |

## Ingestion pipeline (Small-to-Big)

1. **Chunking:** SemanticChunker (split by semantic variation).
2. **Hierarchy:** Child chunks (~150 tokens) are linked to parent chunks (~700 tokens).
3. **Metadata:** Each parent chunk gets a `summary` and `target_question` generated by the OpenAI LLM (optional; can be skipped for faster ingest).

## Retrieval pipeline

1. **Query expansion:** Generate several query variants.
2. **Hybrid search:** Dense vectors + Qdrant native sparse vectors over child chunks (prefetch + RRF) → top 20.
3. **Re-ranking:** Cross-encoder → top 5.
4. **Context:** Fetch the corresponding **parent** chunks for those 5 children to form the context for the LLM.

## Generation

- The prompt instructs the model to answer only from the provided context.
- If the context is insufficient: *"I don't have enough specific information to answer this question."* (or equivalent in the user’s language).
- Inline source citations: `[Source 1]`, `[Source 2]`, etc.

## Usage

### API

```bash
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

**POST /ask**

```json
{
  "question": "Your question here?"
}
```

The retriever chooses the collection automatically. Response fields: `answer`, `sources`, `has_context`, and optionally `collection_name`.

### Streamlit

```bash
streamlit run streamlit_app.py
```

- **Sidebar:** Check API key status; upload PDF/DOCX to ingest; optionally skip LLM metadata for faster ingest; manage database (e.g. clear).
- **Chat:** Enter a question and view the answer with source references.

## Notes

- **PDF/DOCX:** Requires `pypdf` and `python-docx`. DOCX via `unstructured` may need extra system dependencies (e.g. `libmagic`).
- **Multilingual:** OpenAI embeddings and prompts work well for multiple languages; sparse vectors use simple tokenization for keyword match.
- **Qdrant:** Data is stored under `qdrant_db/` (configurable in `config.py` via `persist_dir`). Hybrid search uses native sparse vectors; existing collections need to be re-ingested to get sparse indexing (new ingest creates collections with dense + sparse).
